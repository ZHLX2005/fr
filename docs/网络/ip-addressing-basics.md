## IP 地址与网段

### IP 地址结构

```
IPv4 地址：32 位，4 个字节
表示为点分十进制：192.168.1.100

每个段落 = 8 位 = 0-255
```

### 子网掩码

```
子网掩码决定"哪部分是网络，哪部分是主机"

/24 = 255.255.255.0
     = 前 24 位是网络部分
     = 后 8 位是主机部分

例：192.168.1.100/24
    │──────┬──────│
    网络部分    主机部分
```

### 网段计算

```
网段 192.168.1.0/24：
├─ 网络地址：192.168.1.0 （代表整个网段）
├─ 广播地址：192.168.1.255 （网段内广播）
├─ 第一个可用：192.168.1.1
└─ 最后一个可用：192.168.1.254

可用设备数：254 台
```

### 常见子网前缀

| 前缀 | 子网掩码 | 可用主机数 |
|------|----------|------------|
| /30 | 255.255.255.252 | 2 |
| /29 | 255.255.255.248 | 6 |
| /28 | 255.255.255.240 | 14 |
| /27 | 255.255.255.224 | 30 |
| /26 | 255.255.255.192 | 62 |
| /25 | 255.255.255.128 | 126 |
| **/24** | **255.255.255.0** | **254** |
| /16 | 255.255.0.0 | 65,534 |

## 网段由谁规定？

### DHCP 服务器动态分配

```
┌─────────────────────────────────────────────┐
│  路由器 (通常也是 DHCP 服务器)               │
│                                             │
│  配置 DHCP：                                 │
│  ├─ 网段：192.168.1.0/24                   │
│  ├─ 起始 IP：192.168.1.100                 │
│  ├─ 结束 IP：192.168.1.200                 │
│  └─ 网关：192.168.1.1                      │
└─────────────────────────────────────────────┘
```

### 遵循的标准

| 标准 | 内容 |
|------|------|
| RFC 791 | IP 地址分类和结构 |
| RFC 1812 | IP 子网划分 |
| RFC 2131 | DHCP 协议 |
| RFC 2132 | DHCP 选项 |

### IP 地址分类

| 类别 | 范围 | 用途 |
|------|------|------|
| A 类 | 1.0.0.0 - 126.255.255.255 | 大型网络 |
| B 类 | 128.0.0.0 - 191.255.255.255 | 中型网络 |
| C 类 | 192.0.0.0 - 223.255.255.255 | 小型网络 |
| D 类 | 224.0.0.0 - 239.255.255.255 | 多播 |
| E 类 | 240.0.0.0 - 255.255.255.255 | 保留 |

## 设备如何发现自己的 IP 和网段？

### 方式 1：DHCP 自动获取（最常见）

```
1. 设备连接 WiFi/插入网线
2. 发送 DHCP Discover（广播）
3. DHCP 服务器响应 DHCP Offer
4. 设备发送 DHCP Request
5. DHCP 服务器发送 DHCP Ack（确认）
6. 设备获得：IP、子网掩码、网关、DNS
```

### 方式 2：APIPA（DHCP 不可用时）

```
169.254.0.0/16 范围内的自动地址
叫做"链路本地地址"
只在同一网段内通信，无法上网
```

### 方式 3：手动配置（静态 IP）

```
用户或管理员手动设置：
├─ IP 地址
├─ 子网掩码
├─ 网关
└─ DNS 服务器
```

### 代码示例

```dart
import 'package:network_info_plus/network_info_plus.dart';

Future<void> getNetworkInfo() async {
  final networkInfo = NetworkInfo();

  // 获取 WiFi IP
  final wifiIP = await networkInfo.getWifiIP();
  print('我的 IP: $wifiIP'); // 例如：192.168.1.100

  // 获取子网掩码
  final subnet = await networkInfo.getWifiSubmask();
  print('子网掩码: $subnet'); // 例如：255.255.255.0

  // 获取网关
  final gateway = await networkInfo.getWifiGatewayIP();
  print('网关: $gateway'); // 例如：192.168.1.1
}

// 使用 Dart 网络接口
Future<void> listNetworkInterfaces() async {
  final interfaces = await NetworkInterface.list();
  for (final interface in interfaces) {
    print('接口: ${interface.name}');
    for (final addr in interface.addresses) {
      if (addr.type == InternetAddressType.IPv4) {
        print('  IPv4: ${addr.address}');
      }
    }
  }
}
```

## 网关是什么？

### 网关 = 通往其他网络的"大门"

```
同一网段内通信：
  设备 A：192.168.1.100
  设备 B：192.168.1.101
  → 直接通信，不需要网关

跨网段通信：
  设备 A：192.168.1.100
  设备 C：192.168.2.100
  → 需要网关转发
```

### 家庭网络拓扑

```
Internet
    │
    ▼
┌─────────────┐
│   路由器     │  ← 网关：192.168.1.1
│  (NAT 网关)  │
└──────┬──────┘
       │
       ▼
  ┌────────────────────────────┐
  │      192.168.1.0/24        │
  │                            │
  │  设备1: 192.168.1.100     │
  │  设备2: 192.168.1.101     │
  │  设备3: 192.168.1.102     │
  └────────────────────────────┘
```

## DHCP 详解

### DHCP 消息类型

| 消息 | 方向 | 用途 |
|------|------|------|
| DHCP Discover | 客户端 → 服务器 | 寻找 DHCP 服务器 |
| DHCP Offer | 服务器 → 客户端 | 提供 IP 地址 |
| DHCP Request | 客户端 → 服务器 | 请求使用该 IP |
| DHCP Ack | 服务器 → 客户端 | 确认租约 |
| DHCP Nack | 服务器 → 客户端 | 拒绝（IP 已不再可用） |
| DHCP Release | 客户端 → 服务器 | 释放 IP |
| DHCP Inform | 客户端 → 服务器 | 客户端有静态 IP，请求配置 |

### DHCP 租约

```
DHCP 分配的 IP 有"有效期"

租约时间：24 小时
  → 设备可以无限期使用该 IP
  → 每 12 小时续租一次
  → 设备离开网络后，IP 会被回收

优点：IP 可以被其他设备复用
```

## DNS 是什么？

### DNS = 域名系统

```
没有 DNS：
  访问网站需要记住 IP：142.250.185.78

有 DNS：
  访问网站只需记住域名：www.google.com
  DNS 帮你翻译成 IP
```

### DNS 工作流程

```
用户输入：www.google.com
    │
    ▼
本地 DNS 缓存查询
    │
    ▼
递归 DNS 服务器查询（通常是 ISP 提供）
    │
    ▼
根域名服务器 → .com 域名服务器 → google.com 权威服务器
    │
    ▼
返回 IP：142.250.185.78
    │
    ▼
浏览器使用 IP 建立连接
```

## 一句话总结

> **IP 地址是设备的网络标识，网段由路由器（DHCP）规定。设备通过 DHCP 协议自动获取 IP 和子网掩码，也可以手动配置静态 IP。网关是通往其他网络的大门，DNS 把域名翻译成 IP。**
