# 端到端加密通信设计

## 概述

实现局域网内的端到端加密通信，消息经过中转服务器但流量本身加密，中转服务器无法解密内容。

## 核心原理

```
┌─────────┐                    ┌─────────┐                    ┌─────────┐
│  设备 A  │                    │  中转   │                    │  设备 B  │
│         │                    │  服务器  │                    │         │
│ (公钥A) │                    │ (无密钥) │                    │ (公钥B) │
│ (私钥A) │                    │         │                    │ (私钥B) │
└────┬────┘                    └────┬────┘                    └────┬────┘
     │                              │                              │
     │  1. 交换公钥                 │                              │
     │ ──────────────────────────> │                              │
     │                              │                              │
     │                              │  2. 交换公钥                │
     │                              │ <─────────────────────────── │
     │                              │                              │
     │  3. 用 B 的公钥加密消息       │                              │
     │ ────────────────────────────────────────────────────────>   │
     │         (中转服务器只看到密文)                              │
     │                              │                              │
     │                              │                              │
     │  4. 用 A 的公钥加密回复       │                              │
     │ <───────────────────────────────────────────────────────    │
     │         (中转服务器只看到密文)                              │
```

## 密钥交换流程

### 方案一：Diffie-Hellman 密钥交换

```
设备 A:  生成 DH 私钥 a，公钥 A = g^a mod p
设备 B:  生成 DH 私钥 b，公钥 B = g^b mod p

A → B: 发送 A
B → A: 发送 B

设备 A: 计算共享密钥 K = B^a mod p
设备 B: 计算共享密钥 K = A^b mod p

K 是对称加密密钥，双方持有相同
```

### 方案二：非对称加密（更简单）

```
设备 A: 生成 RSA/EC 密钥对 (公钥A, 私钥A)
设备 B: 生成 RSA/EC 密钥对 (公钥B, 私钥B)

交换公钥后：
A → B: 用 B 的公钥加密消息
B → A: 用 A 的公钥加密回复
```

## 消息格式设计

```dart
class EncryptedMessage {
  final String senderId;      // 发送者设备 ID
  final String receiverId;   // 接收者设备 ID
  final String ciphertext;    // 用接收者公钥加密的密文
  final String encryptedKey; // 用接收者公钥加密的对称密钥（如果用混合加密）
  final int timestamp;       // 时间戳
  final String nonce;         // 随机数（防重放）
}
```

## 混合加密方案（推荐）

结合非对称加密和对称加密的优点：

```
发送消息时：
1. 生成随机对称密钥 K
2. 用 K 加密消息内容 (AES)
3. 用接收者公钥加密 K (RSA/ECIES)
4. 发送 [加密的K + 加密的消息]

接收消息时：
1. 用私钥解密 K
2. 用 K 解密消息内容
```

## 中转服务器职责

| 职责 | 说明 |
|------|------|
| 消息转发 | 只负责存储转发密文 |
| 公钥分发 | 协助设备间交换公钥（可选） |
| 在线状态 | 记录设备是否在线 |
| **不参与** | 解密、加密、密钥生成 |

## 安全特性

| 特性 | 实现方式 |
|------|----------|
| 机密性 | 非对称/对称加密 |
| 完整性 | 消息摘要/HMAC |
| 认证 | 数字签名 |
| 前向保密 | 临时密钥 (ECDH) |
| 防重放 | 时间戳 + nonce |
