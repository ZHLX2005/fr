# Line Demo 设置页面重设计

## 概述

将线游戏的速度设置从底部弹出面板改为全新页面，左右分栏布局：左侧控制速度，右侧实时演示下落+炸开动画。极简线条主义风格，全主题色适配。

## 导航

- **方式**：`Navigator.push` 进入独立页面
- **返回按钮**：页面左上角，`Icons.arrow_back_ios_new`，主题色 + 小尺寸，与游戏页返回按钮风格一致
- **进入/退出动画**：复用现有水动画 `_WaterExitPainter`，从设置图标点击时遮罩进入，返回时遮罩退出
- **游戏状态**：进入设置页时暂停游戏（保存快照），返回时倒计时恢复

## 布局

左右分栏，左侧 40% / 右侧 60%。

```
┌───────────────────────────────────┐
│ ←                                 │
│                                   │
│   ┌─────────┐   ┌──────────────┐  │
│   │ 控制区   │   │   演示区      │  │
│   │         │   │              │  │
│   │ 标签     │   │  ┌────────┐  │  │
│   │ 数值     │   │  │ 圆框内  │  │  │
│   │ 滑块     │   │  │ 下落    │  │  │
│   │ 快/慢    │   │  │ +炸开   │  │  │
│   │         │   │  └────────┘  │  │
│   └─────────┘   └──────────────┘  │
│                                   │
└───────────────────────────────────┘
```

## 左侧：控制区

垂直居中，从上到下：

1. **标签**：`下落速度`，`theme.textTheme.labelSmall`，`onSurfaceVariant` 色
2. **数值**：`2500ms`，`fontWeight: w100`，大字号（约 32rpx），`fontFeature: tabularFigures`，主色低透明度
3. **横向滑块**：复用 `_LineThumbShape`（极小实心圆点 + 细线轨道 1.5px），方向反转——左快右慢
4. **两端标签**：`快` / `慢`，`labelSmall`，`onSurfaceVariant`

滑块拖动实时更新右侧演示动画的下落速度。

## 右侧：演示区

圆角线框内模拟游戏场景：

- **外框**：`borderRadius: 16rpx`，`border: 1px solid primary.withAlpha(0.2)`
- **内部元素**：
  - 3 条竖线（与游戏一致，低透明度）
  - 1 条判定线（75% 位置，低透明度）
  - 1 个圆圈从顶部下落到判定线附近 → 炸开 → 消失 → 新圆圈重新下落（循环）
  - **只演示中间列**，简洁不杂乱
- **动画**：独立的 `AnimationController`，不与游戏共享状态。进入页面即开始，退出 dispose
- **速度同步**：下落 duration 跟随左侧滑块 `_dropDurationMs` 实时变化

## 配色

所有元素使用 `theme.colorScheme.primary`，透明度 0.2~0.4，与游戏页极简线条风格保持一致。

## 实现要点

- 设置页为独立 Widget（`_SpeedSettingsPage`），通过 `Navigator.push` 导航
- 演示动画复用 `_GamePainter` 的绘制逻辑（圆圈 + 炸开粒子），但使用独立的数据和 controller
- 进入/返回时的游戏暂停/恢复逻辑复用现有 `_pausedSnapshots` + `_startCountdown` 机制
- 返回时通过 `Navigator.pop()` 回到游戏页，触发倒计时恢复
