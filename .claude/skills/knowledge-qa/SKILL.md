# knowledge-qa 工作流程

## 工作流程

### 1. 理解主题

明确用户想要了解的主题，将主题名称作为文档标题。

### 2. 深度搜索 (使用mmx)

执行3-5次并行搜索，覆盖不同角度：

```bash
# 主搜索 - 基础概念
mmx search query --q "<主题> 基础知识 原理 概念" --output json --quiet

# 进阶搜索 - 实践应用
mmx search query --q "<主题> 实践 应用 案例" --output json --quiet

# 细节搜索 - 技术细节
mmx search query --q "<主题> 技术细节 实现 原理" --output json --quiet

# 最佳实践
mmx search query --q "<主题> 最佳实践 教程 指南" --output json --quiet
```

### 3. 分析搜索结果

从搜索结果中提取关键信息：
- 核心概念和定义
- 工作原理
- 实践方法
- 常见问题和解决方案
- 参考资源

### 4. 编写/更新文档

**去冗余原则**：
- 同一主题只创建**一个**文档
- 如果 docs/temp/ 中已存在相关主题的文档，**更新**而非创建新的
- 只有当主题明显不同时才创建新文档

**文件操作原则**：
- **优先使用 Edit 工具**：更新现有文档的特定部分
- **少用 Write 工具**：仅在文档不存在时使用
- **不重复创建**：先 `ls docs/temp/` 检查是否已有相关文档

**文档结构**：

```markdown
# <主题>知识指南

## 概述
<2-3句话概括主题>

## 核心概念
<关键概念定义>

## 工作原理
<原理解释>

## 实践应用
<应用场景和方法>

## 常见问题
<FAQ>

## 参考资源
<搜索到的优质链接>
```

### 5. 输出位置

文档输出到：`docs/temp/<主题>-guide.md`

文件名使用英文或拼音，避免特殊字符。

### 6. 验证

```bash
# 检查文档是否存在
ls -la docs/temp/<主题>-guide.md
```

## 错误处理

| 错误 | 处理方式 |
|------|----------|
| 搜索无结果 | 尝试不同关键词，或告知用户 |
| 文档已存在 | 直接更新该文档，不创建新的 |
| 写入失败 | 检查目录权限，确认 docs/temp/ 存在 |
