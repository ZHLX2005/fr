## 1. Animal 枚举：等级、力量、走子优先级

```java
// 来源: src/model/piece/animal/Animal.java:3-36
public enum Animal {
    TRAPPED_ANIMAL(0, 0, "Trapped Animal"),
    RAT(500, 4, "Rat"),
    CAT(200, 8, "Cat"),
    DOG(300, 7, "Dog"),
    WOLF(400, 6, "Wolf"),
    LEOPARD(500, 5, "Leopard"),
    TIGER(800, 2, "Tiger"),
    LION(900, 1, "Lion"),
    ELEPHANT(1000, 3, "Elephant");

    private final int piecePower;       // AI 评估子力价值
    private final int movePriority;     // AI 走子排序（越小越优先）
    private final String pieceName;
}
```

三个维度务必区分：

| 枚举 | `ordinal`(=attackRank，决定吃子层级) | `piecePower`(AI 子力价值) | `movePriority`(AI 走子排序) | 中文 |
|---|---|---|---|---|
| `TRAPPED_ANIMAL` | 0 | 0 | 0 | 陷阱中降级态 |
| `RAT` | 1 | **500** | 4 | 鼠 |
| `CAT` | 2 | 200 | 8 | 猫 |
| `DOG` | 3 | 300 | 7 | 狗 |
| `WOLF` | 4 | 400 | 6 | 狼 |
| `LEOPARD` | 5 | 500 | 5 | 豹 |
| `TIGER` | 6 | 800 | 2 | 虎 |
| `LION` | 7 | 900 | 1 | 狮 |
| `ELEPHANT` | 8 | 1000 | 3 | 象 |

> **attackRank = 枚举 ordinal**：构造棋子时 `super(Animal.XXX, ..., Animal.XXX.ordinal(), ...)`（如 `Rat.java:21`）。**piecePower ≠ rank**：鼠的 power=500（与豹并列），因鼠能克象、战略价值高。

## 3. 棋子类层级

```
Piece (抽象)
├── CommonPiece  (抽象) ← Cat / Dog / Wolf / Leopard   标准走法
├── SpecialPiece (抽象) ← Lion / Tiger                 可跳河
└── (直接继承 Piece)
    ├── Rat        可入河 + 鼠克象 + 水陆边界
    └── Elephant   不克鼠
```

> ⚠️ **源码缺陷**：`Rat.java:24-25` 与 `Elephant.java:22-24` 的 **3 参构造器**（带 `isFirstMove`）误把 `pieceType` 写成 `Animal.CAT`：
> ```java

>     super(Animal.CAT, pieceCoordinate, pieceColor, Animal.CAT.ordinal(), isFirstMove);
> ```
> 实际运行走 2 参构造器（`isFirstMove` 恒 true），不影响游戏，但移植时勿照抄。

## 4. 吃子规则（核心）

### 4.1 通用规则：attackRank ≥ defenseRank
`CommonPiece` 与 `SpecialPiece` 共用同一判定：

```java
// 来源: src/model/piece/CommonPiece.java:36-41  (SpecialPiece.java:75-80 同形)
final Piece pieceAtDestination = potentialDestinationTerrain.getPiece();
final PlayerColor pieceAtDestinationColor = pieceAtDestination.getPieceColor();
if (this.pieceColor != pieceAtDestinationColor
        && this.pieceAttackRank >= pieceAtDestination.pieceDefenseRank) {
    validMoves.add(new CaptureMove(board, this, potentialDestinationCoordinate, pieceAtDestination));
}
```

- **等级相同也可吃**（`>=`，非 `>`）。
- **陷阱中棋子 defenseRank=0**（见 `01-棋盘模型` §5），故任意攻击方（哪怕鼠 rank1）都能吃陷阱中的任意敌方棋子。

### 4.2 特例一：鼠克象（鼠能吃象）
鼠（rank1）本不该吃象（rank8），但规则特例允许。`Rat` 硬编码可吃目标集合：

```java
// 来源: src/model/piece/animal/Rat.java:59-61
if (this.pieceColor != pieceAtDestinationColor
        && (pieceAtDestination.getPieceDefenseRank() == Animal.ELEPHANT.ordinal()   // 克象
         || pieceAtDestination.getPieceDefenseRank() == Animal.RAT.ordinal()        // 吃同级鼠
         || pieceAtDestination.getPieceDefenseRank() == Animal.TRAPPED_ANIMAL.ordinal())) { // 吃陷阱棋子(rank0)
```

即鼠**只能**吃：象（特例）、另一鼠（同级）、陷阱中棋子（rank0）。鼠**不能**吃猫/狗/狼/豹/虎/狮（通用规则下 rank1 吃不了更高 rank，且不在特例集合里）。

### 4.3 特例二：象不克鼠
象（rank8）通用规则下能吃一切，但特例排除鼠：

```java
// 来源: src/model/piece/animal/Elephant.java:53-58
final Piece pieceAtDestination = potentialDestinationTerrain.getPiece();
if (this.pieceColor != pieceAtDestinationColor
        && pieceAtDestination.getPieceDefenseRank() != Animal.RAT.ordinal()) {  // 不能吃非陷阱鼠
    validMoves.add(new Move.CaptureMove(board, this, potentialDestinationCoordinate, pieceAtDestination));
}
```

> **细节**：若鼠正处象的陷阱中（defenseRank=0 ≠ RAT.ordinal()=1），象**可以**吃——符合"陷阱中谁都能吃"。即象只对"非陷阱中的鼠"免疫。

### 4.4 特例三：鼠的水陆边界
鼠吃子额外受"水陆同介质"约束：

```java
// 来源: src/model/piece/animal/Rat.java:62-64
if (BoardUtils.isLand(this.pieceCoordinate) && !BoardUtils.isRiver(potentialDestinationCoordinate)   // 陆→陆
        || BoardUtils.isRiver(this.pieceCoordinate) && BoardUtils.isRiver(potentialDestinationCoordinate)) { // 河→河
    validMoves.add(new CaptureMove(board, this, potentialDestinationCoordinate, pieceAtDestination));
}
```

即：**鼠在河里不能吃岸上的棋子**（反之岸上鼠不能吃河中鼠）。河中鼠 vs 河中鼠可互吃。这与 README "Rat cannot capture from water" 一致。

## 5. 移动规则（4 类差异）

所有棋子共用偏移量 `{-7, -1, +1, +7}`（=上、左、右、下）+ 列边界排除。

### 5.1 CommonPiece（Cat/Dog/Wolf/Leopard）
```java
// 来源: src/model/piece/CommonPiece.java:24-46
for (final int currentPotentialOffset : POTENTIAL_MOVE_COORDINATES) {  // {-7,-1,1,7}
    int potentialDestinationCoordinate = this.pieceCoordinate + currentPotentialOffset;
    if (BoardUtils.isInBoundary(potentialDestinationCoordinate)
            && !BoardUtils.isRiverOrDen(potentialDestinationCoordinate, this.pieceColor)) { // 禁入河/己穴
        if (isColumnZeroExclusion(...) || isColumnSixExclusion(...)) continue;
        final Terrain potentialDestinationTerrain = board.getTerrain(potentialDestinationCoordinate);
        if (!potentialDestinationTerrain.isTerrainOccupied()) {
            validMoves.add(new StandardMove(board, this, potentialDestinationCoordinate));  // 走空格
        } else {
            // §4.1 吃子判定
        }
    }
}
```
普通 1 格直行；**禁入河、禁入己方兽穴**。

### 5.2 SpecialPiece（Lion/Tiger，可跳河）
基本同 CommonPiece，但当**相邻格是河**时触发跳跃逻辑（详见 `03-走法生成` §跳河算法）：
- **纵向跳**：河高 3 行 → 循环检查 3 个河格是否被占（被鼠阻挡则放弃）→ 落到对岸第 4 格（`+4*±7`）。
- **横向跳**：河宽 2 列 → 检查 2 个河格 → 落到第 3 格（`+3*±1`）。

```java
// 来源: src/model/piece/SpecialPiece.java:33-52 (纵向跳片段)
if ((currentPotentialOffset == -7 || currentPotentialOffset == 7)
        && BoardUtils.isRiver(potentialDestinationCoordinate)) {
    boolean isRiverTerrainOccupied = false;
    for (int i = 0; i < 3; i++) {                                    // 跨 3 个河格
        potentialDestinationTerrain = board.getTerrain(tempPotentialDestinationCoordinate);
        if (potentialDestinationTerrain.isTerrainOccupied()) { isRiverTerrainOccupied = true; break; }
        tempPotentialDestinationCoordinate += currentPotentialOffset;
    }
    if (!isRiverTerrainOccupied) {
        if (BoardUtils.isInBoundary(tempPotentialDestinationCoordinate)) {
            potentialDestinationCoordinate = tempPotentialDestinationCoordinate;  // 落对岸
            potentialDestinationTerrain = board.getTerrain(tempPotentialDestinationCoordinate);
        }
    } else { continue; }   // 被鼠阻挡，放弃该方向
}
```
狮/虎**不能**踏入河（只能跳越）；落点若为敌子且满足吃子条件则 CaptureMove。

### 5.3 Rat（可入河）
```java
// 来源: src/model/piece/animal/Rat.java:47-49
if (BoardUtils.isInBoundary(potentialDestinationCoordinate)
        && !BoardUtils.isDen(potentialDestinationCoordinate, this.pieceColor)) {  // 仅排除己穴，不排除河→可入河
```
鼠可入河；吃子逻辑见 §4.2/§4.4。

### 5.4 Elephant
同 CommonPiece（`isRiverOrDen` 排除河/己穴，禁入河）；吃子见 §4.3（不克鼠）。

## 6. 总结：吃子判定决策树（移植参考）

```
攻击方 A 欲走向格 D（有敌方棋子 B）：
├─ B 在敌方陷阱? → defenseRank=0 → 任意 A 可吃（含鼠吃狮）     [Board.Builder.setPiece]
├─ A 是 Rat?
│   ├─ B 是 Elephant 或 Rat(同级) → 可吃，但需满足水陆同介质    [Rat.java:59-64]
│   └─ 否则 → 不可吃
├─ A 是 Elephant?
│   └─ B 是 非陷阱鼠? → 不可吃；否则可吃                       [Elephant.java:56]
└─ 其他（Cat/Dog/Wolf/Leopard/Lion/Tiger）
    └─ A.attackRank >= B.defenseRank? → 可吃                     [CommonPiece.java:39]
```

> 移植要点：把 `defenseRank` 做成 **getter**（`isEnemyTrap ? 0 : attackRank`）而非可变字段，比 Java 版的"重建时 setPiece"更干净、无状态漂移。
