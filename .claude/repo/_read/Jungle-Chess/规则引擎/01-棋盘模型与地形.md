## 1. 棋盘尺寸与一维坐标

斗兽棋棋盘为 **9 行 × 7 列 = 63 格**，用**一维数组**索引。

```java
// 来源: src/model/board/BoardUtils.java:11-13
public static final int START_TERRAIN_INDEX = 0;
public static final int NUM_TERRAINS = 63;
public static final int NUM_TERRAINS_PER_ROW = 7;
```

坐标换算：`index = row * 7 + col`，`row ∈ [0,8]`，`col ∈ [0,6]`。**行 0 在顶部（红方），行 8 在底部（蓝方）**。

边界判定：

```java
// 来源: src/model/board/BoardUtils.java:25-27
public static boolean isInBoundary(final int coordinate) {
    return coordinate >= 0 && coordinate < 63;
}
```

### 列边界（防跨行）
因一维索引，左右移动 ±1 会跨行。用两个预计算布尔数组排除：

```java
// 来源: src/model/board/BoardUtils.java:14-15
public static final boolean[] COLUMN_ZERO = populateColumn(0);  // 第 0 列：不能 -1（左）
public static final boolean[] COLUMN_SIX = populateColumn(6);   // 第 6 列：不能 +1（右）

// 来源: src/model/board/BoardUtils.java:68-75
private static boolean[] populateColumn(int columnIndex) {
    final boolean[] column = new boolean[NUM_TERRAINS];
    do {
        column[columnIndex] = true;
        columnIndex += NUM_TERRAINS_PER_ROW;   // 每 7 格一列
    } while (columnIndex < NUM_TERRAINS);
    return column;
}
```

所有棋子移动都套用 `isColumnZeroExclusion / isColumnSixExclusion`（见各 `determineValidMoves`，如 `CommonPiece.java:48-54`）。

### 代数记谱（调试/存档用）
```java
// 来源: src/model/board/BoardUtils.java:85-96
"a9","b9","c9","d9","e9","f9","g9",   // row 0 = rank 9（红方）
"a8", ... "g8",                        // row 1
...
"a1","b1","c1","d1","e1","f1","g1"     // row 8 = rank 1（蓝方）
```
即国际象棋式记谱：行 0 = rank 9（顶/红），行 8 = rank 1（底/蓝），列 a–g（col 0–6）。`getPositionAtCoordinate / getCoordinateAtPosition` 做双向映射（`:207-213`）。

## 2. Board 数据结构（不可变）

```java
// 来源: src/model/board/Board.java:14-35
public class Board {
    private final List<Terrain> chessboard;        // 63 格
    private final Map<Integer, Piece> boardConfig; // 坐标→棋子
    private final Collection<Piece> bluePieces;    // 蓝方存活棋子
    private final Collection<Piece> redPieces;     // 红方存活棋子
    private final BluePlayer bluePlayer;
    private final RedPlayer redPlayer;
    private final Player currentPlayer;            // 当前手方
    private final Move transitionMove;             // 链向上一步（悔棋/历史）

    private Board(Builder builder) {
        this.chessboard = constructChessboard(builder);
        this.boardConfig = Collections.unmodifiableMap(builder.boardConfig);
        this.bluePieces = determineActivePieces(this.chessboard, PlayerColor.BLUE);
        this.redPieces = determineActivePieces(this.chessboard, PlayerColor.RED);
        final Collection<Move> blueValidMoves = determineValidMoves(this.bluePieces);
        final Collection<Move> redValidMoves  = determineValidMoves(this.redPieces);
        this.bluePlayer = new BluePlayer(this, blueValidMoves, redValidMoves);
        this.redPlayer  = new RedPlayer(this, blueValidMoves, redValidMoves);
        this.currentPlayer = getPlayer(builder.nextMovePlayer, this.bluePlayer, this.redPlayer);
        this.transitionMove = builder.transitionMove != null ? builder.transitionMove : Move.MoveFactory.getNullMove();
    }
}
```

**关键设计**：Board **不可变（immutable / functional）**——每走一步生成一个全新 Board（`Move.execute()` 见 `03-走法生成`）。构造时即算出双方合法走法（`determineValidMoves` `:104-110`）。

`Terrain` 用空对象模式区分空格/占格：

```java
// 来源: src/model/board/Terrain.java:25-27, 37-80
public static Terrain constructTerrain(int terrainCoordinate, Piece piece) {
    return piece != null ? new OccupiedTerrain(terrainCoordinate, piece)
                         : EMPTY_TERRAINS.get(terrainCoordinate);  // 复用空对象
}
// EmptyTerrain.toString() = "**"；OccupiedTerrain.toString() = 棋子 2 字符代码（红小写/蓝大写）
```

## 4. 地形规则

### 4.1 河流 River（12 格，两块 3×2 矩形）

```java
// 来源: src/model/board/BoardUtils.java:35-42
public static boolean isRiver(final int coordinate) {
    return (coordinate == 22 || coordinate == 23 ||     // 左河 col1-2 row3
            coordinate == 29 || coordinate == 30 ||     // 左河 col1-2 row4
            coordinate == 36 || coordinate == 37 ||     // 左河 col1-2 row5
            coordinate == 25 || coordinate == 26 ||     // 右河 col4-5 row3
            coordinate == 32 || coordinate == 33 ||     // 右河 col4-5 row4
            coordinate == 39 || coordinate == 40);      // 右河 col4-5 row5
}
```

- **左河**：col 1-2，row 3-5（6 格）；**右河**：col 4-5，row 3-5（6 格）。
- 规则：**仅鼠可入河**；狮/虎可跳越（见 `02-棋子体系`）；其余棋子禁止入河（`CommonPiece`/`Elephant` 用 `isRiverOrDen` 排除）。

### 4.2 兽穴 Den（每方 1 格）

```java
// 来源: src/model/board/BoardUtils.java:44-52
public static boolean isDen(final int coordinate, PlayerColor pieceColor) {
    if (pieceColor.isBlue()) return (coordinate == 59);   // 蓝穴 d1（行8 col3）
    else if (pieceColor.isRed()) return (coordinate == 3); // 红穴 d9（行0 col3）
    else return false;
}
```

规则：**棋子不能进入己方兽穴**（`!isDen(dest, ownColor)`）。敌方棋子进入对方兽穴即获胜（判定见 `BluePlayer.isDenPenetrated` / `RedPlayer.isDenPenetrated`）。

### 4.3 陷阱 Trap（每方 3 格）

"敌方陷阱"= 对方兽穴周围的 3 格，**按棋子颜色相对判定**：

```java
// 来源: src/model/board/BoardUtils.java:58-66
public static boolean isEnemyTrap(final int coordinate, PlayerColor pieceColor) {
    if (pieceColor.isBlue()) {                // 蓝棋看红方陷阱
        return (coordinate == 2 || coordinate == 4 || coordinate == 10);
    } else if (pieceColor.isRed()) {          // 红棋看蓝方陷阱
        return (coordinate == 52 || coordinate == 58 || coordinate == 60);
    } else return false;
}
```

- 蓝棋的敌方（红）陷阱 = `{2, 4, 10}`（红穴 d9 周围：c9, e9, d8）
- 红棋的敌方（蓝）陷阱 = `{52, 58, 60}`（蓝穴 d1 周围：d2, c1, e1）
- 规则：棋子**进入敌方陷阱后防御等级降为 0**（见 `02-棋子体系` §吃子），可被任意敌方棋子吃掉；己方陷阱对己方棋子无影响。

### 4.4 普通陆地辅助判定
```java
// 来源: src/model/board/BoardUtils.java:29-33
public static boolean isLand(final int coordinate) {
    return (!isRiver(coordinate)
            && !isDen(coordinate, BLUE) && !isDen(coordinate, RED)
            && !isEnemyTrap(coordinate, BLUE) && !isEnemyTrap(coordinate, RED));
}
```
（鼠的水陆边界吃子规则用到，见 `02-棋子体系`。）

## 5. Builder 与陷阱降级（核心机制入口）

```java
// 来源: src/model/board/Board.java:236-244
public Builder setPiece(final Piece piece) {
    this.boardConfig.put(piece.getPieceCoordinate(), piece);
    if (BoardUtils.isEnemyTrap(piece.getPieceCoordinate(), piece.getPieceColor())) {
        piece.setPieceDefenseRank(0);                          // 在敌方陷阱→防御降 0
    } else {
        piece.setPieceDefenseRank(piece.getPieceAttackRank()); // 否则=攻击等级
    }
    return this;
}
```

**这是陷阱机制的总开关**：每当棋盘被（重新）构建，落子时即判定该棋子是否在敌方陷阱上，据此设置 `defenseRank`。因每步走子都经 Builder 重建棋盘（`Move.execute()`），陷阱状态自动随位置更新——进入陷阱→防御0，离开→恢复。

> 详见 `02-棋子体系与吃子规则.md`（defenseRank 如何参与吃子判定）与 `03-走法生成与走子执行.md`（execute 如何重建棋盘）。
