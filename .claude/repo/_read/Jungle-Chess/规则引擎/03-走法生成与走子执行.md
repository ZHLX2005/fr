## 2. MoveFactory：从坐标构造走法

```java
// 来源: src/model/board/Move.java:220-241
public static class MoveFactory {
    public static Move getNullMove() { return NULL_MOVE; }

    public static Move createMove(final Board board, final int currentCoordinate, final int destinationCoordinate) {
        for (final Move move : board.getAllValidMoves()) {           // 在全部合法走法中匹配
            if (move.getCurrentCoordinate() == currentCoordinate
                    && move.getDestinationCoordinate() == destinationCoordinate) {
                return move;
            }
        }
        return NULL_MOVE;                                            // 找不到→非法
    }

    public static Move createBannedRepetitiveMove(final Board board, final int currentCoordinate, final int destinationCoordinate) {
        return new BannedRepetitiveMove(board, board.getPiece(currentCoordinate), destinationCoordinate);
    }
}
```

**合法性本质**：`createMove` 在 `board.getAllValidMoves()`（蓝+红双方全部预计算走法，`Board.java:197-202`）中查找 from→to；找不到即 `NullMove`（非法）。

## 3. 走子执行（不可变重建）

```java
// 来源: src/model/board/Move.java:53-61
public Board execute() {
    final Board.Builder builder = new Board.Builder();
    // 当前方：保留除"动子"外的全部棋子
    this.board.getCurrentPlayer().getActivePieces().stream()
              .filter(piece -> !this.movedPiece.equals(piece))
              .forEach(builder::setPiece);
    // 对方：全部保留
    this.board.getCurrentPlayer().getEnemyPlayer().getActivePieces().forEach(builder::setPiece);
    // 动子落到目标格（movePiece 返回新实例）
    builder.setPiece(this.movedPiece.movePiece(this));
    builder.setNextMovePlayer(this.board.getCurrentPlayer().getEnemyPlayer().getAllyColor()); // 换手
    builder.setMoveTransition(this);                                // 链向上一步
    return builder.build();
}
```

**核心**：每步生成**全新 Board**（functional style）。`movePiece` 由各动物子类实现，返回同种动物在目标坐标的新实例（如 `Rat.movePiece` `Rat.java:35-37`）。因重建时 `Builder.setPiece` 重算陷阱状态（见 `01-棋盘模型` §5），陷阱降级自动随位置更新。

悔棋：
```java

public Board undo() {
    final Board.Builder builder = new Board.Builder();
    this.board.getAllPieces().forEach(builder::setPiece);          // 还原全部棋子
    builder.setNextMovePlayer(this.board.getCurrentPlayer().getAllyColor());
    return builder.build();
}
```

## 4. MoveTransition / MoveStatus：走法结果

```java
// 来源: src/model/player/Player.java:29-35
public MoveTransition makeMove(final Move move) {
    if (!isMoveValid(move)) {                                       // 不在合法走法集
        return new MoveTransition(this.board, this.board, move, MoveStatus.INVALID_MOVE);
    }
    final Board transitionBoard = move.execute();
    return new MoveTransition(this.board, transitionBoard, move, MoveStatus.DONE);
}
```

```java
// 来源: src/model/board/MoveStatus.java:3-17
public enum MoveStatus {
    DONE { @Override public boolean isDone() { return true; } },
    INVALID_MOVE { @Override public boolean isDone() { return false; } };

}
```

`MoveTransition`（`MoveTransition.java:3-31`）封装 `fromBoard / toBoard / transitionMove / moveStatus`。AI 与 Controller 据 `moveStatus.isDone()` 判断是否执行成功。

## 5. 跳河算法详解（SpecialPiece）

接 `02-棋子体系` §5.2，完整跳河逻辑：

```java
// 来源: src/model/piece/SpecialPiece.java:22-85
private final int[] POTENTIAL_MOVE_COORDINATES = {-7, -1, 1, 7};

for (final int currentPotentialOffset : POTENTIAL_MOVE_COORDINATES) {
    int potentialDestinationCoordinate = this.pieceCoordinate + currentPotentialOffset;
    int tempPotentialDestinationCoordinate = potentialDestinationCoordinate;
    if (BoardUtils.isInBoundary(potentialDestinationCoordinate)
            && !BoardUtils.isDen(potentialDestinationCoordinate, this.pieceColor)) {
        if (isColumnZeroExclusion(...) || isColumnSixExclusion(...)) continue;
        Terrain potentialDestinationTerrain = board.getTerrain(potentialDestinationCoordinate);

        // —— 纵向跳（±7）：相邻格是河才触发 ——
        if ((currentPotentialOffset == -7 || currentPotentialOffset == 7)
                && BoardUtils.isRiver(potentialDestinationCoordinate)) {
            boolean isRiverTerrainOccupied = false;
            for (int i = 0; i < 3; i++) {                           // 河高 3 行 → 检查 3 格
                potentialDestinationTerrain = board.getTerrain(tempPotentialDestinationCoordinate);
                if (potentialDestinationTerrain.isTerrainOccupied()) { isRiverTerrainOccupied = true; break; }
                tempPotentialDestinationCoordinate += currentPotentialOffset;
            }
            if (!isRiverTerrainOccupied && BoardUtils.isInBoundary(tempPotentialDestinationCoordinate)) {
                potentialDestinationCoordinate = tempPotentialDestinationCoordinate;       // 落到 +4*offset
                potentialDestinationTerrain = board.getTerrain(tempPotentialDestinationCoordinate);
            } else { continue; }                                    // 被鼠阻挡 → 放弃
        }
        // —— 横向跳（±1）：同理，循环 2 次（河宽 2 列 → 落到 +3*offset）——
        if ((currentPotentialOffset == -1 || currentPotentialOffset == 1) && BoardUtils.isRiver(...)) {
            for (int i = 0; i < 2; i++) { /* 同上 */ }
            ...
        }
        // —— 落点判定 ——
        if (!potentialDestinationTerrain.isTerrainOccupied()) {
            validMoves.add(new Move.StandardMove(board, this, potentialDestinationCoordinate));
        } else {
            final Piece pieceAtDestination = potentialDestinationTerrain.getPiece();
            if (this.pieceColor != pieceAtDestination.getPieceColor()
                    && this.pieceAttackRank >= pieceAtDestination.pieceDefenseRank) {
                validMoves.add(new Move.CaptureMove(board, this, potentialDestinationCoordinate, pieceAtDestination));
            }
        }
    }
}
```

**跳跃距离**：
- 纵向：从河岸格 `+4*7`（跨 3 河格，落到对岸）。例：虎 @coord15（行2 col1）向下 → 越过 22/29/36 → 落 coord43（行6 col1）。
- 横向：`+3*1`（跨 2 河格）。例：狮 @coord21（行3 col0）向右 → 越过 22/23 → 落 coord24（行3 col3）。

**被鼠阻挡**：越过任一河格若有棋子（实际只可能是鼠）→ `isRiverTerrainOccupied=true` → `continue`（该方向无走法）。

## 6. 三回重复禁手

```java
// 来源: src/model/board/BoardUtils.java:110-145
public static void checkThreeFoldRepetition(MoveLog currentPlayerMoveLog) {
    if (currentPlayerMoveLog.size() < 6) return;
    // 看己方 MoveLog 倒数第 2/4/6 步（间隔对手走子）
    final Piece last2MovesPiece = currentPlayerMoveLog.getMove(size - 2).getMovedPiece();
    final Piece last4MovesPiece = currentPlayerMoveLog.getMove(size - 4).getMovedPiece();
    final Piece last6MovesPiece = currentPlayerMoveLog.getMove(size - 6).getMovedPiece();
    if (last2MovesPiece.equalsTypeTwo(last4MovesPiece) && last4MovesPiece.equalsTypeTwo(last6MovesPiece)) {
        // 同一棋子
        final int last2 = ...getDestinationCoordinate(); final int last4 = ...; final int last6 = ...;
        if (last2 == last4 && last4 == last6) {                    // 走到同一格
            if (clearRepetitionMoveLog(currentPlayerMoveLog)) {     // 双方都在重复→清空 MoveLog
                ... return;
            }
            // 否则把"该子→该格"走法替换为 BannedRepetitiveMove
            List<Move> newMoves = new ArrayList<>();
            for (Move old : oldValidMoves) {
                if (old.getDestinationCoordinate() != last2) newMoves.add(old);
                else newMoves.add(Move.MoveFactory.createBannedRepetitiveMove(...));
            }
            GameFrame.get().getChessBoard().getCurrentPlayer().setValidMoves(unmodifiableList(newMoves));
        }
    }
}
```

- 检测**同一玩家连续 3 次把同一棋子走到同一格**（倒数 2/4/6 步：同子 + 同终点）。
- 命中则把该走法替换为 `BannedRepetitiveMove`（临时禁手），直到该方走出别的位置。
- `clearRepetitionMoveLog`（`:147-164`）检查对手倒数 3/5/7 步——若**双方都在重复**，直接清空双方 MoveLog。
- `BannedRepetitiveMove.execute()` 返回原棋盘（`:202-204`，no-op），AI（`Minimax.java:49` 等）与合法走法生成会跳过它。

> ⚠️ **架构缺陷**：此方法直接改 `GameFrame.get().getChessBoard().getCurrentPlayer().setValidMoves(...)`——规则引擎耦合 Swing 单例（`BoardUtils.java:130, 142`）。移植时须把 MoveLog/validMoves 作为参数传入，解除 UI 耦合。

## 7. 终局判定

### 7.1 标准胜负（模型层）
```java
// 来源: src/model/board/BoardUtils.java:185-189
public static boolean isGameOverScenarioStandardConditions(final Board board) {
    return board.getCurrentPlayer().isDenPenetrated()                          // 兽穴被入侵
            || board.getCurrentPlayer().getActivePieces().isEmpty()            // 棋子被吃光
            || board.getCurrentPlayer().getValidMoves().isEmpty();             // 无棋可走（困毙=输）
}
```

### 7.2 特殊模式胜负（耦合 UI）

```java
private static boolean isGameOverScenarioSpecialGameModeConditions() {
    return GameFrame.get().getPlayerPanel().getBlueCurrentTimerSecondsBlitzMode() == 0   // blitz 超时
```
