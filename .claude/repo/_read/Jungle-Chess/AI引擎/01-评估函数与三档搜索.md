## 1. 接口与评估总公式

```java
// 来源: src/model/artificialintelligence/StandardBoardEvaluator.java:9-25
public final class StandardBoardEvaluator implements BoardEvaluator {
    private final static int DEPTH_MULTIPLIER = 10;
    private final static int MOBILITY_MULTIPLIER = 5;
    private final static int CAPTURE_MOVES_MULTIPLIER = 1;
    private static final int ENEMY_RUNNING_OUT_OF_PIECES_MULTIPLIER = 25_000;
    private static final int ENEMY_RUNNING_OUT_OF_VALID_MOVES_MULTIPLIER = 25_000;
    private final static int ENEMY_DEN_PENETRATED_MULTIPLIER = 50_000;

    @Override
    public int evaluate(final Board board, final int depth) {
        return scorePlayer(board.bluePlayer(), depth) - scorePlayer(board.redPlayer(), depth);  // 零和
    }
}
```

**净分 = scorePlayer(蓝) − scorePlayer(红)**。正=蓝优，负=红优。蓝方取最大、红方取最小（见 §3）。

`scorePlayer` = 4 分量之和：
```java
// 来源: StandardBoardEvaluator.java:27-33
private int scorePlayer(final Player player, final int depth) {
    return pieceValue(player, depth)
         + mobility(player, depth)
         + captureMoves(player)
         + isEnemyDenPenetrated(player, depth);
}
```

## 2. 四个评估分量

### 2.1 pieceValue（子力 + 位置 + 敌方无子奖励）
```java
// 来源: StandardBoardEvaluator.java:51-61
private static int pieceValue(final Player player, int depth) {
    int pieceValueScore = 0;
    for (final Piece piece : player.getActivePieces()) {
        pieceValueScore += (piece.getPiecePower() + piece.positionDevelopmentScore()); // 子力 + 位置分
    }
    int enemyOutOfPiecesBonus = 0;
    if (player.getEnemyPlayer().getActivePieces().isEmpty()) {
        enemyOutOfPiecesBonus = ENEMY_RUNNING_OUT_OF_PIECES_MULTIPLIER * depthBonus(depth); // 吃光奖励
    }
    return pieceValueScore + enemyOutOfPiecesBonus;
}
```
`piecePower`（Animal 枚举，鼠500…象1000）+ `positionDevelopmentScore`（查位置表，见 §5）。

### 2.2 mobility（机动性 + 敌方无走法奖励）
```java
// 来源: StandardBoardEvaluator.java:63-75
private static int mobility(Player player, int depth) {
    if (enemy 无走法 && enemy 无子)  return 25_000 * depthBonus(depth);
    if (enemy 无走法 && enemy 有子)  return 2 * 25_000 * depthBonus(depth);   // 困毙=大优
    return MOBILITY_MULTIPLIER * mobilityRatio(player);
}
private static int mobilityRatio(Player player) {
    return (int) ((player.getValidMoves().size() * 10.0f) / player.getEnemyPlayer().getValidMoves().size());
}
```

### 2.3 captureMoves（即时吃子机会）
```java
// 来源: StandardBoardEvaluator.java:77-85
private static int captureMoves(final Player player) {
    int captureScore = 0;
    for (final Move move : player.getValidMoves()) {
        if (move.isCaptureMove()) {
            captureScore += move.getCapturedPiece().getPiecePower();   // 累加可吃子的 power
        }
    }
    return captureScore * CAPTURE_MOVES_MULTIPLIER;                    // ×1
}
```

### 2.4 isEnemyDenPenetrated（入穴终局）
```java
// 来源: StandardBoardEvaluator.java:87-89
private static int isEnemyDenPenetrated(Player player, int depth) {
    return player.getEnemyPlayer().isDenPenetrated() ? ENEMY_DEN_PENETRATED_MULTIPLIER * depthBonus(depth) : 0;
}
```

### depthBonus（越浅的胜利分越高）
```java
// 来源: StandardBoardEvaluator.java:91-93
private static int depthBonus(int depth) {
    return depth == 0 ? 1 : DEPTH_MULTIPLIER * depth;   // 促使 AI 优先速胜
}
```

## 3. 三档搜索 Bot

三者都实现 `MoveStrategy`（`execute(board) → Move`）。蓝方最大化、红方最小化。

### 3.1 BOT1：Minimax（深度 4）
```java
// 来源: src/model/artificialintelligence/Minimax.java:37-86 (execute 主体)
for (final Move move : board.getCurrentPlayer().getValidMoves()) {
    if (!move.isBannedRepetitiveMove()) {
        final MoveTransition moveTransition = board.getCurrentPlayer().makeMove(move);
        if (moveTransition.getMoveStatus().isDone()) {
            currentValue = board.getCurrentPlayer().getAllyColor().isBlue()
                    ? min(moveTransition.getToBoard(), this.searchDepth - 1)    // 蓝方：对手取 min
                    : max(moveTransition.getToBoard(), this.searchDepth - 1);   // 红方：对手取 max
            if (蓝 && currentValue >= highestSeenValue) { highestSeenValue = currentValue; optimalMove = move; }
            else if (红 && currentValue <= lowestSeenValue)  { lowestSeenValue  = currentValue; optimalMove = move; }
        }
    }
}
```
递归 `max/min`（`:88-146`）：`depth==0` 或 `isGameOverScenarioStandardConditions` 时返回 `evaluator.evaluate(board, depth)`；否则遍历合法走法取 max/min。**无剪枝**，深度 4 ≈ 26 万节点。

### 3.2 BOT2：α-β + MoveOrdering（深度 6）
```java
// 来源: src/model/artificialintelligence/AlphaBetaWithMoveOrdering.java:96-116 (max)
public int max(final Board board, final int depth, final int highest, final int lowest) {
    if (depth == 0 || BoardUtils.isGameOverScenarioStandardConditions(board)) {
        this.numEvaluatedBoards++; return this.evaluator.evaluate(board, depth);
    }
    int currentHighest = highest;                                    // alpha
    for (final Move move : MoveSorter.STANDARD.sort(board.getCurrentPlayer().getValidMoves())) {
        final MoveTransition moveTransition = board.getCurrentPlayer().makeMove(move);
        if (moveTransition.getMoveStatus().isDone()) {
            currentHighest = Math.max(currentHighest,
                    min(moveTransition.getToBoard(), depth - 1, currentHighest, lowest));
            if (currentHighest >= lowest) { this.cutOffsProduced++; return currentHighest; }  // β 剪枝
        }
    }
    return currentHighest;
}
```
根节点用 `MoveSorter.EXPENSIVE` 排序（`:46`），递归内用 `STANDARD`。`currentHighest >= lowest` 剪枝并计 `cutOffsProduced`。根节点搜到致胜走（入穴/敌无子/敌无走法）立即 `break`（`:58-71`）。深度 6 ≈ 1 亿节点。

### 3.3 BOT3：+ Quiescence（动态深度）
在 BOT2 基础上，深度=1 时测"战术活跃度"决定是否延伸：

```java
// 来源: src/model/artificialintelligence/PruningOrderingQuiescenceSearch.java:158-175
private int calculateQuiescenceDepth(final Board toBoard, final int depth) {
    if (depth == 1 && this.quiescenceCount < MAX_QUIESCENCE) {       // MAX_QUIESCENCE = 5000*5
        int activityMeasure = 0;
        if (BoardUtils.getIntoEnemyTrap(toBoard.getTransitionMove())) activityMeasure += 1;  // 进敌陷阱
        for (final Move move : BoardUtils.lastNMoves(toBoard, 2)) {
            if (move.isCaptureMove()) activityMeasure += 1;          // 近 2 步含吃子
        }
        if (activityMeasure >= 2) { this.quiescenceCount++; return 2; }  // 延伸到深度 2
    }
    return depth - 1;
}
```
缓解 horizon effect：吃子/进陷阱等战术活跃时延伸搜索深度。`max/min`（`:101-147`）递归调 `calculateQuiescenceDepth` 替代固定 `depth-1`。

### 性能实测（README §Performance Testing）
20 个测试局面、双方轮流执红蓝、共 120 局：BOT2 vs BOT1 = 17胜22和1负；BOT3 vs BOT2 = 8胜21和11负（Quiescence 未能显著超越 BOT2，README 归因于 BOT3 在吃子序列上"犹豫"导致和棋）。

## 4. MoveSorter（提升 α-β 剪枝率）

```java
// 来源: src/model/artificialintelligence/MoveSorter.java:14-35
public enum MoveSorter {
    STANDARD {   // 递归内用
        public Collection<Move> sort(final Collection<Move> moves) {
            return Ordering.from((move1, move2) -> ComparisonChain.start()
                    .compareTrueFirst(move1.isCaptureMove(), move2.isCaptureMove())               // 1. 先吃子
                    .compare(move1.getMovedPiece().getPieceType().getMovePriority(),             // 2. 再按 movePriority
                             move2.getMovedPiece().getPieceType().getMovePriority())
                    .result()).immutableSortedCopy(moves);
        }
    },
    EXPENSIVE {  // 根节点用
        public Collection<Move> sort(final Collection<Move> moves) {
            return Ordering.from(... ComparisonChain.start()
                    .compareTrueFirst(getIntoEnemyDen(move1), getIntoEnemyDen(move2))            // 1. 进敌方兽穴
                    .compareTrueFirst(getIntoEnemyTrapWithoutEnemyNearby(move1), ...)            // 2. 进无敌方附近的敌陷阱
                    .compareTrueFirst(move1.isCaptureMove(), move2.isCaptureMove())              // 3. 吃子
                    .compare(movePriority, movePriority)                                         // 4. movePriority
                    .result()).immutableSortedCopy(moves);
        }
    };
}
```

`getIntoEnemyTrapWithoutEnemyNearby`（`:45-108`）：硬编码每个陷阱坐标（蓝方看 2/4/10，红方看 52/58/60）周围是否有敌方棋子，判断"进陷阱是否安全"——若安全则优先该走法。

## 5. 位置评估表（PlayerColor）

```java
// 来源: src/model/player/PlayerColor.java:178-188 (示例：蓝鼠位置表)
private static final int[] BLUE_RAT_PREFERRED_POSITIONS = {
    11, 13, 50, 1000, 50, 13, 13,   // 行0 (rank9, 红方区) — 1000 在 col3(红穴 d9)
    11, 12, 13, 50, 13, 13, 13,
    ...
    8, 8, 8, 0, 8, 8, 8              // 行8 (rank1, 蓝方己区) — 0 在 col3(蓝穴 d1, 己穴禁入)
};
```

每种动物 × 每色各一张 63 元素表（`BLUE/RED_{RAT,CAT,DOG,WOLF,LEOPARD,TIGER,LION,ELEPHANT}_PREFERRED_POSITIONS`，共 16 张）。`positionDevelopmentScore()` 按 `pieceColor.xxxDevelopmentScore(coordinate)` 查表：

```java
// 来源: src/model/piece/animal/Rat.java:40-42
@Override public int positionDevelopmentScore() {
    return this.pieceColor.ratDevelopmentScore(this.pieceCoordinate);
}
```

**语义**：
- **越靠近敌方兽穴分越高**（敌方穴格 = 1000、邻格 = 50）——鼓励进攻。
- **0 = 该格禁入/不利**：常见于河列（col 1-2/4-5 的行 3-5），非鼠棋子不应靠近河；以及己方兽穴列。
- 红表 = 蓝表 180° 旋转（镜像上下），策略对双方对称。
- 强子（狮/虎/象）位置分基数更高（25-30 量级），弱子（鼠/猫）较低（8-15）。

> 移植：16 张表直接作为 `const List<int>` 搬运，零成本；它们是 AI 棋力的核心"经验"。

---

## 6. 移植要点

1. **AI 放 Isolate**：深度 6 ≈ 1 亿节点，必须后台线程 + 可取消。
2. **评估表常量化**：16 张 63 元素 `const` 数组直接搬。
3. **α-β 优先**：BOT2（α-β+MoveOrdering）性价比最高；Quiescence 收益不明显（实测），可作可选增强。
4. **depthBonus**：保留"越浅胜利分越高"机制，促使 AI 速胜而非拖延。
5. **解耦**：`MoveSorter.getIntoEnemyTrapWithoutEnemyNearby` 硬编码坐标，移植时改为查 `isEnemyTrap` + 邻格遍历，更通用。
