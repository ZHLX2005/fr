## 1. 数据模型（`web/types/game.ts`）

### 1.1 枚举
```typescript
// 来源: web/types/game.ts:1-15
export enum PieceColor { BLUE, RED }              // BLUE=0, RED=1
export enum PieceAnimal {
    RAT, CAT, DOG, WOLF, CHEETAH, TIGER, LION, ELEPHANT   // 0..7（豹此处叫 CHEETAH）
}
```
> **注意**：`PieceAnimal` 的数值即等级（RAT=0 … ELEPHANT=7），吃子比较直接用 `<`（见 §3）。

### 1.2 核心接口
```typescript
// 来源: web/types/game.ts:17-58
export interface PieceType {
    animal: PieceAnimal;
    position: BoardPosition;
    color: PieceColor;
}

export interface Board {
    width: number;          // 7
    height: number;         // 9
    turns: PieceColor[];    // [BLUE, RED]，回合顺序
    turn: PieceColor;       // 当前手方
    game_ended: boolean;
    objects: {
        traps: BoardObject[];   // 陷阱
        ends: BoardObject[];    // 兽穴
        water: BoardObject[];   // 河流
    },
    pieces: PieceType[];
}

export interface BoardPosition { x: number; y: number; }   // x=列[0,6], y=行[0,8]

export interface BoardObject {
    type: BoardObjectType;          // END / TRAP / WATER
    position: BoardPosition;
    color: PieceColor | null;       // 兽穴/陷阱有归属色；水无色
}
```

> ⚠️ **移植陷阱 1**：`PieceType` **无唯一 ID**（`game.ts:17-21`）。React 用 `getPieceSource(piece)`（如 "BR"）作 key（`Piece.tsx:35`），但同色同种动物不重复时勉强可行；Dart 移植建议加 `id` 字段，避免 key 冲突与状态追踪困难。
>
> ⚠️ **移植陷阱 2**：`Board.turn` 类型注解写 `PieceColor`（`:31`），但 `loadBoard` 实际赋值 `turn: 0`（`loadBoard.ts:64`，**数字 index** 而非枚举）。`page.tsx:72` 又用 `board.turn < board.turns.length - 1 ? board.turn + 1 : 0` 做数字递增。运行时靠 JS 弱类型蒙混，Dart 强类型下须统一为 `int turnIndex` 或 `PieceColor turn`。

### 地形（`loadObjects` `:24-53`）
- **陷阱 traps**：红 `{(2,0),(4,0),(3,1)}`，蓝 `{(3,7),(2,8),(4,8)}`（`color` 标归属）。
- **兽穴 ends**：红 `(3,0)`、蓝 `(3,8)`。
- **河流 water**：12 格，`(1,3)(2,3)(4,3)(5,3)` + `(1,4)(2,4)(4,4)(5,4)` + `(1,5)(2,5)(4,5)(5,5)`（左河 col1-2 / 右河 col4-5，行 3-5）。

## 3. 规则引擎（`web/helpers/game/getPosibleMoves.ts`，纯函数）

119 行纯函数，输入 `board/pieces/piece/activeCell`，输出合法落点 `BoardPosition[]`。

```typescript
// 来源: web/helpers/game/getPosibleMoves.ts:52-119
export default function getPosibleMoves(board, pieces, piece, activeCell): BoardPosition[] {
    if (piece == undefined || pieces == undefined) return [];
    const positions: BoardPosition[] = [];
    let activePiece = getActivePiece(pieces, activeCell);
    if (activePiece == null) return [];

    const directions = [[1, 0], [-1, 0], [0, 1], [0, -1]];     // 4 方向（右/左/下/上）

    for (let direction of directions) {
        const newPosition = { x: activePiece.position.x + direction[0], y: activePiece.position.y + direction[1] };

        // —— 狮/虎跳河 ——
        if (piece.animal == PieceAnimal.TIGER || piece.animal == PieceAnimal.LION) {
            const ratPositions = getRatPositions(pieces);
            while (positionHasWater(board, newPosition.x, newPosition.y)) {   // 逐格越过水
                if (isRatBlockingWaterJump(ratPositions, newPosition)) break; // 被鼠阻挡
                newPosition.x += direction[0];
                newPosition.y += direction[1];
            }
        }

        // —— 非鼠不能入水 ——
        if (piece.animal != PieceAnimal.RAT)
            if (positionHasWater(board, newPosition.x, newPosition.y)) continue;

        // —— 吃子判定 ——
        const pieceAboutToEat = getPieceAboutToEat(pieces, newPosition);
        const otherIsVulnerable = positionHasATrap(board, newPosition.x, newPosition.y);
        if (pieceAboutToEat && !otherIsVulnerable) {            // ⚠️ 目标在陷阱→跳过守卫（见 §5 偏差）
            if (piece.color == pieceAboutToEat.color) continue; // 同色不吃

            const elephantToRat = piece.animal == ELEPHANT && pieceAboutToEat.animal == RAT;
            if (elephantToRat) continue;                        // 象不克鼠

            const ratToElephant = piece.animal == RAT && pieceAboutToEat.animal == ELEPHANT;
            const hasLessValue = piece.animal < pieceAboutToEat.animal;  // 等级比较（枚举数值）
            if (!ratToElephant && hasLessValue) continue;       // 低 rank 吃不了高 rank（鼠克象例外）

            const pieceIsRat = piece.animal == RAT;
            const isInWater = positionHasWater(board, piece.position.x, piece.position.y);
            const otherPieceIsInWater = positionHasWater(board, newPosition.x, newPosition.y);
            if (pieceIsRat && isInWater && !otherPieceIsInWater) continue; // 鼠出水不吃岸
        }

        // —— 不能进己方兽穴 ——
        const end = getEndInPosition(board, newPosition.x, newPosition.y);
        if (end && end.color === piece.color) continue;

        if (isInBounds(board, newPosition.x, newPosition.y)) positions.push(newPosition);
    }
    return positions;
}
```

辅助函数（同文件）：
- `isInBounds`（`:3-5`）：`x∈[0,width), y∈[0,height)`。
- `positionHasWater/positionHasATrap`（`:7-13`）：在 `board.objects.water/traps` 中 `some` 查。
- `getEndInPosition`（`:15-18`，**导出**）：查兽穴。
- `getActivePiece`（`:20-28`）：按坐标找选中棋子。
- `getRatPositions`（`:30-38`）/ `isRatBlockingWaterJump`（`:40-45`）：狮虎跳河的鼠阻挡判定。

### 与 Jungle-Chess 规则对照
| 规则 | TochuGV | Jungle-Chess | 一致? |
|---|---|---|---|
| 4 方向 1 格 | ✅ `directions` | ✅ offsets {-7,-1,1,7} | ✅ |
| 狮虎跳河 + 被鼠阻挡 | ✅ while 循环 | ✅ for 循环 | ✅ |
| 非鼠禁入水 | ✅ | ✅ | ✅ |
| 象不克鼠 | ✅ `elephantToRat` | ✅ | ✅ |
| 鼠克象 | ✅ `ratToElephant` | ✅ | ✅ |
| 等级吃子（rank≥） | ✅ `<` 取反 | ✅ `>=` | ✅ |
| 鼠出水不吃岸 | ✅ | ✅ | ✅ |
| 陷阱降级 | ⚠️ 偏差（见 §5） | ✅ defenseRank=0 | ❌ |
| 不进己穴 | ✅ | ✅ | ✅ |

## 4. 胜负（`web/helpers/game/whoWon.ts`）

```typescript
// 来源: web/helpers/game/whoWon.ts:3-15
export default function whoWon(board: Board): string {
    let index: number = board.turns.findIndex(t => t == board.turn) - 1;   // ⚠️ 取"上一回合"
    switch (index % board.turns.length) {
        case PieceColor.BLUE: return "Blue";
        case PieceColor.RED:  return "Red";
        default: return "";
    }
}
```

**怪味**：胜负判定依赖 `turn` **已被推进到下一手**（`page.tsx:72` 在走子时先 `turn+1`）。故胜方 = `turns[当前turnIndex - 1]`。`index % length` 处理 wrap（如 BLUE=0 时 `-1 % 2 = -1`，JS 中为 -1 → default 返回空——**潜在 bug**，边界情况可能判不出胜方）。

胜负触发点在 `page.tsx:48-54`：走到敌方 `end` 格即 `game_ended=true` + 弹 Modal。

> ⚠️ **规则缺失**：TochuGV **仅判定"入敌方兽穴"胜负**，**无困毙判定、无和棋（回合上限）、无超时、无 AI**（`clock_time`/`GameModeEnum` 在 `game.ts:23-33` 仅注释预留）。完整规则须从 Jungle-Chess 移植。

## 5. ⚠️ 规则偏差清单（移植 TochuGV 逻辑时必须修正）

| # | 位置 | 偏差 | 正确规则（Jungle-Chess） | 修正 |
|---|---|---|---|---|
| 1 | `getPosibleMoves.ts:89-90` | `otherIsVulnerable`（目标在陷阱）时**跳过整个吃子守卫块**，导致同色棋子在自己陷阱上也可被"吃"（page.tsx filter 会移除己方子） | 陷阱中棋子防御降 0，**任意敌方**可吃，但同色仍不可吃 | 改为：目标在敌方陷阱→defense=0；仍校验 `color !=` |
| 2 | `getPosibleMoves.ts:11-13` | `positionHasATrap` 不区分陷阱**归属色**（己方陷阱也触发） | 仅**敌方陷阱**降级 | 加 `trap.color != piece.color` 判定 |
| 3 | `whoWon.ts:4` | `findIndex(...) - 1`，BLUE 先手胜时 `-1 % 2` JS=-1→判空 | 胜方=刚走子方 | 用显式 `lastMover` 而非 `turn-1` |
| 4 | 全局 | 无困毙/无和棋/无超时/无 AI | Jungle-Chess 有完整 5 条件 + 3 档 AI | 从 Jungle-Chess 移植 |
| 5 | `page.tsx:72` | `turn` 类型混用（注解 PieceColor / 赋值数字） | 强类型 | Dart 统一 `int turnIndex` |

> **结论**：TochuGV 的 `getPosibleMoves.ts` 结构干净（4 方向 + 跳河 + 吃子分支清晰），适合作 Dart 规则引擎的**骨架**，但 §5 的 5 处偏差必须修正——**建议直接以 Jungle-Chess 的完整规则为准**，仅借鉴 TochuGV 的函数拆分风格。
